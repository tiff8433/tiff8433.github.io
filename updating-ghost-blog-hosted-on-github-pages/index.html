Updating Ghost Blog Hosted on Github Pages

30 Jan 2016

This post will describe the workflow need to write a blogpost in the ghost editor and deploy to github pages so that it is viewable at http://yourgithubusername.github.io.

If you need help getting Github Pages to serve your Ghost blog, the following links, that utilize buster, are helpful:
1. http://ledtechnica.com/free-ghost-hosting-on-github-pages/
2. http://briank.im/i-see-ghosts/
3. http://blog.sunnyg.io/2015/09/24/ghost-with-github/

Now that you have your ghost-github-buster blog set up with, how do you create and deploy a new post?

  1. open your terminal and cd into your ghost folder
  2. npm start to start the server
  3. open you browser and go to localhost:<port number>/ghost
  4. sign in, if prompted
  5. the ghost editor should open up, click on 'New Post'
  6. start writing
  7. when you are finished, go to the 'SAVE DRAFT' dropdown menu and click on 'Publish Now'
  8. cd into your ghost folder that contains the 'static' folder, not into the 'static' folder
  9. type buster generate to regenerate the static files
  10. type buster deploy and your pages will be added, committed and pushed to github
  11. your blog should be updated shortly

Until you have gone through the workflow a couple of times, it may not be very intuitive on how to update your post. Hopefully the above steps will be a quick reminder, until it becomes second nature.
