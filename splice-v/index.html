Splice vs Slice

25 Jan 2016

Remembering when to use the slice() vs splice() can be confusing, probably because they are both array methods and they sound so much alike. There are some key differences:

  1. What is returned:
    • slice: returns a new array with selected elements
    • splice: adds or removes elements from an array and returns those removed elements
  2. Mutates original array:
    • slice: No. Slice creates a new array with the returned elements. The original array remains intact.
    • splice: Yes. Splice will change the original array.
  3. Arguments:
    • slice: There are 2 arguments, as you can see here: array.slice(start, end)
      • 'start' is a required integer that indicates the index where you'd like to begin the copy
      • 'end' is an optional integer that indicates the end of a selection.
    • splice: There are 2 required arguments for splice and optional additional parameters: array.splice(start, howMany, option1, option2, option3...)
      • 'start' indicates which index to ad or remove items
      • 'howMany' determines how many items to remove. It is given a value of 0 if you want to select to the end of the array
      • optional additional items can be included in the arguments as elements to be added into the array
  4. Negative Integers:
    • slice: Use a negative integer as a parameter to select from the end of the array
    • splice: If the start index is a negative number, it will specify the position from the end of the array.

I have listed the main differences between two widely used array methods. In a future post, we will go through some examples.
